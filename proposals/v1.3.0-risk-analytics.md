# Proposal: v1.3.0 — Risk Analytics

## Intent

Build Risk Analytics without schema migrations, new entities, or new dependencies — pure additive change reusing existing calculator/service/repository patterns.

## Scope

### In Scope

| Phase | Metric | Data Source | Complexity |
|-------|--------|-------------|------------|
| 1 | Max Consecutive Wins/Losses | Reuse `compute_streaks()` — already computed in equity endpoint | O(n) done |
| 1 | Avg Holding Time | `entry_datetime` / `exit_datetime` — per-trade duration already in `get_detail()` | O(n) |
| 1 | Holding Time Distribution | Same, bucketed: `<1h`, `1-4h`, `4-24h`, `1-7d`, `>7d` | O(n) |
| 1 | Avg Risk per Trade | `risk_amount` — count trades with/without populated risk | O(n) |
| 1 | Risk Utilization | `risk_amount / (position_size * \|entry_price - stop_loss\|) * 100` | O(n) |
| 3 | Exposure by Asset | `asset_id` + position_size × entry_price × direction — reuse breakdown pattern | O(n) |
| 3 | Exposure by Session | `market_session_id` → join MarketSession catalog | O(n) |
| 3 | Exposure by Strategy | `strategy_id` — reuse `breakdown_by_strategy()` pattern | O(n) |
| 3 | Cross-asset Correlation | Pearson on per-trade PnL by asset pair — min 5 trades per pair | O(n·m) |
| 4 | Risk of Ruin | `( (1-edge) / (1+edge) )^(capital/avg_risk)` — O(1) formula | O(1) |
| 4 | Kelly Fraction | `(p·b - q) / b` — O(1) formula | O(1) |
| 4 | Expectancy (variance-adj) | `expectancy / std(pnl)` — Sharpe-like | O(n) |

### Out of Scope (v1.4.0+)

- **Phase 2** — MAE, MFE, Exit Efficiency: blocked — needs intra-trade high/low price data. Schema change required.

## Endpoints

| Method | Path | Body | Degradation |
|--------|------|------|-------------|
| GET | `/analytics/risk-metrics` | All Phase 1 + Phase 4 | null per metric, never 500 |
| GET | `/analytics/exposure/by-asset` | Exposure by asset | Omit pairs with <1 trade |
| GET | `/analytics/exposure/by-session` | Exposure by session | Skip missing session_ids |
| GET | `/analytics/exposure/by-strategy` | Exposure by strategy | Same as breakdown |
| GET | `/analytics/exposure/correlation` | Cross-asset Pearson | Min 5 trades threshold |

### Decision: New endpoint, not extended summary

`GET /analytics/risk-metrics` is **new** — does NOT bloat `GET /analytics/summary`. Separates executive KPIs from specialized risk analysis.

**Calculator pattern** — same as existing:
```python
def compute_avg_holding_time(trades: list[Trade]) -> float | None:
    durations = [duration_hours(t) for t in trades if t.exit_datetime]
    return sum(durations) / len(durations) if durations else None
```

### Capital for Risk of Ruin / Kelly

Read from `Account.balance` via UoW. If `None` or account not found → return null only for affected metrics.

### Degradation Rule (NEW)

- Missing `risk_amount` → `avg_risk_per_trade = None`, `risk_utilization = None`
- Missing `stop_loss` → `risk_utilization` skips trade (smaller N)
- Missing capital → `risk_of_ruin = None`, `kelly_fraction = None`
- Empty trade list → all metrics return null
- **Never return 500.** Never fail the whole request for one missing field.

### Exposure: Open Trade Inclusion

`list_closed()` excludes open trades. For Phase 3 (exposure), we need open positions too. Two options:

| Option | Effort | Risk |
|--------|--------|------|
| Add `list_all()` to TradeRepository | Low — ~10 lines | Low — pure additive |
| Parameterize `list_closed(status=None)` | Low — changes existing signature | Moderate — may affect callers |

**Recommendation: New `list_all()` method** — safer, additive pattern.

### Cross-asset Correlation

- Minimum 5 trades per asset pair before computing correlation
- Early bail-out: `len(pair_trades) < 5 → skip`
- Returns `list[CorrelationPair(asset_a, asset_b, pearson, n_trades)]`

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| `risk_amount` sparsely populated → degraded metrics | Medium | Report `trades_without_risk` counter in response |
| Capital not configured | Medium | null for Ruin/Kelly — graceful degradation |
| Correlation on large datasets | Low | Min 5-trade threshold per pair; O(n·m) with early bail-out |
| Open trades excluded from exposure | Low | New `list_all()` method on repository |

## Rollback Plan

Revert the commit. No migrations, no schema changes, no data affected.
